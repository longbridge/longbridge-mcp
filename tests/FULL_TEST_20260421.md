## 序列化管道升级（2026-04-21 后补修复 — C.1 架构）

上游 unix-seconds 字符串现在输出为 RFC3339。架构分两层：

**pipeline 层**（`src/serialize/`）只处理稳定惯例，范围刻意收窄：

- `classify_field` 仅将 `*_at` 后缀识别为 Timestamp（Rust/Longbridge 上游惯例）；不额外维护字段名白名单，避免误分类。
- `TimestampSerializer::serialize_str` 支持字符串形式的 unix 秒：若可解析为 [946684800, 4102444800]（约 2000–2100）的 i64，则转 RFC3339；超出范围或非数字保持原样，从而保留 `"0"` / `"-62135596800"` 等哨兵语义。
- `TimestampSerializer::serialize_seq` 返回 `TimestampSeq`，用于 `created_at:[unix,...]` 之类罕见形态（虽然现阶段少见，保留架构一致）。

**per-tool 层**（`src/tools/`）对非 `_at` 字段显式声明 path：

- 新增 `serialize::convert_unix_paths(value, paths)`：path 语法 `a.b.c` / `*`（匹配数组元素或 map 值），原地替换命中的 unix 秒字符串。
- 新增 `http_client::http_get_tool_unix(client, path, params, unix_paths)`：流程 `transform_json`（snake_case + `_at`）→ 解析 Value → `convert_unix_paths` → 序列化。path 基于**已 snake_case 后**的字段名。
- `market_status` 手工构造 Value，直接调 `convert_unix_paths` 后走 `tool_json`。
- `institution_rating` 将两段响应拼合后调用 `convert_unix_paths`。
